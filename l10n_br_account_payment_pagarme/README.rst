Este módulo implementa a integração do gateway de pagamento Pagar.me com o sistema de ordens de pagamento brasileiro do Odoo, fornecendo suporte completo para:

- **Cartões de Crédito e Débito**: Processamento seguro com suporte a parcelamento
- **Boleto Bancário**: Geração automática de boletos com códigos de barras
- **PIX**: Pagamentos instantâneos com QR codes
- **Anti-fraude**: Análise automática de risco
- **Split de Pagamentos**: Divisão automática entre recebedores
- **Webhooks**: Atualizações automáticas de status
- **Assinaturas**: Pagamentos recorrentes (futuro)

Configuração
============

Configuração do Pagar.me
-------------------------

1. **Criar Conta no Pagar.me**:
   
   - Acesse https://pagar.me e crie sua conta
   - Complete o processo de verificação KYC
   - Obtenha suas chaves de API (Secret Key e Public Key)

2. **Configurar Provedor de Pagamento**:

   Vá em **Faturamento > Configuração > Provedores de Pagamento > Pagar.me**

   Configure os seguintes campos obrigatórios:

   - **API Key**: Sua chave secreta do Pagar.me
   - **Public Key**: Sua chave pública para criptografia client-side
   - **Sandbox Mode**: Habilitado para testes, desabilitado para produção

3. **Configurar Métodos de Pagamento**:

   - **Cartão de Crédito**: Configure parcelamento máximo e valor mínimo por parcela
   - **Boleto**: Configure prazo de vencimento e instruções
   - **PIX**: Configure tempo de expiração e tipo de chave PIX

4. **Configurar CNAB**:

   Vá em **Faturamento > Configuração > Configuração CNAB**

   - **Processador CNAB**: Selecione "Pagar.me"
   - Configure as mesmas credenciais da API
   - Defina método de pagamento padrão

5. **Configurar Webhooks**:

   Use o botão "Configurar Webhooks" na configuração CNAB para registrar automaticamente os webhooks no Pagar.me.

Uso
===

Processamento de Pagamentos
---------------------------

1. **Criar Fatura**:
   
   - Configure o modo de pagamento com configuração CNAB Pagar.me
   - A fatura será automaticamente integrada com o Pagar.me

2. **Processar Ordem de Pagamento**:

   - Confirme a ordem de pagamento
   - O sistema criará automaticamente as transações no Pagar.me
   - Status será atualizado via webhooks

3. **Acompanhar Status**:

   - Use o botão "Verificar Status" para consultar atualizações manuais
   - Webhooks atualizarão automaticamente o status das transações

Métodos de Pagamento Suportados
-------------------------------

**Cartão de Crédito/Débito**:

- Bandeiras: Visa, Mastercard, Elo, Hipercard, American Express, Diners
- Parcelamento: Até 24x (configurável)
- Captura: Automática ou manual
- 3D Secure: Disponível para maior segurança

**Boleto Bancário**:

- Bancos: Todos os principais bancos brasileiros
- Vencimento: Configurável (padrão 3 dias)
- Código de barras: Gerado automaticamente
- Instruções: Personalizáveis

**PIX**:

- Chaves: CPF, CNPJ, email, telefone ou aleatória
- QR Code: Gerado automaticamente
- Expiração: Configurável (padrão 60 minutos)
- Confirmação: Instantânea via webhook

API do Pagar.me
===============

Endpoints Principais
-------------------

**Base URL**:
- Sandbox: ``https://api.pagar.me/core/v5``
- Produção: ``https://api.pagar.me/core/v5``

**Autenticação**:

.. code-block:: python

   import base64
   
   api_key = "sk_test_abc123..."
   encoded_key = base64.b64encode(f"{api_key}:".encode()).decode()
   headers = {
       "Authorization": f"Basic {encoded_key}",
       "Content-Type": "application/json"
   }

**Endpoints Utilizados**:

1. **Criar Transação**:
   
   ``POST /transactions``

   .. code-block:: json

      {
        "amount": 10000,
        "currency": "brl",
        "payment_method": "credit_card",
        "customer": {
          "name": "João Silva",
          "email": "joao@exemplo.com",
          "document": "12345678901",
          "document_type": "cpf"
        },
        "credit_card": {
          "installments": 3,
          "capture": true
        }
      }

2. **Consultar Transação**:
   
   ``GET /transactions/{transaction_id}``

3. **Criar Boleto**:

   .. code-block:: json

      {
        "amount": 5000,
        "payment_method": "boleto",
        "boleto": {
          "due_at": "2024-12-31T23:59:59Z",
          "instructions": "Pagar somente com este boleto"
        }
      }

4. **Criar PIX**:

   .. code-block:: json

      {
        "amount": 2500,
        "payment_method": "pix",
        "pix": {
          "expires_in": 3600
        }
      }

Webhooks
--------

**Eventos Suportados**:

- ``transaction.paid`` - Transação aprovada
- ``transaction.refused`` - Transação recusada
- ``transaction.refunded`` - Transação estornada
- ``charge.paid`` - Cobrança paga
- ``charge.failed`` - Cobrança falhou

**Estrutura do Webhook**:

.. code-block:: json

   {
     "id": "hook_abc123",
     "type": "transaction.paid",
     "created_at": "2024-01-15T10:30:00Z",
     "data": {
       "id": "tran_def456",
       "status": "paid",
       "amount": 10000,
       "currency": "brl"
     }
   }

**Validação de Assinatura**:

.. code-block:: python

   import hmac
   import hashlib
   
   def validate_webhook(payload, signature, secret):
       expected = hmac.new(
           secret.encode(),
           payload.encode(),
           hashlib.sha256
       ).hexdigest()
       return hmac.compare_digest(signature, f"sha256={expected}")

Tratamento de Erros
------------------

**Códigos de Erro Comuns**:

- ``001``: Erro geral
- ``002``: Valor da transação inválido
- ``003``: Dados do cartão inválidos
- ``004``: Cartão expirado
- ``005``: Saldo insuficiente
- ``006``: Transação não autorizada

**Retry Logic**:

O módulo implementa retry automático para:
- Timeouts de rede
- Erros temporários da API
- Rate limiting

Segurança
=========

Criptografia
------------

- **TLS 1.2+**: Todas as comunicações usam HTTPS
- **Chaves Assimétricas**: Public key para criptografia client-side
- **HMAC SHA-256**: Validação de assinatura de webhooks
- **Tokenização**: Dados de cartão nunca armazenados

Conformidade
-----------

- **PCI DSS**: Compliance total através do Pagar.me
- **LGPD**: Dados pessoais minimizados e criptografados  
- **BCB**: Compatível com regulamentações do Banco Central

Anti-fraude
----------

- **Análise em Tempo Real**: Score de risco para cada transação
- **Machine Learning**: Detecção de padrões suspeitos
- **Regras Customizadas**: Configuração específica por merchant
